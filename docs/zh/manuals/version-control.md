# 版本控制

Defold 项目非常适合与 [Git](https://git-scm.com) 搭配使用，但同步操作需在编辑器外部进行。请使用您偏好的 Git 客户端或命令行执行克隆、获取、拉取、提交、推送、创建分支和解决冲突。

## 已更改的文件

当项目目录是至少包含一次提交的 Git 工作树根目录时，Defold 会在 *Changed Files* 编辑器面板中列出检测为已添加、已修改、已删除或已重命名的未忽略文件。它通过将磁盘上的文件直接与当前提交（`HEAD`）比较来生成这些条目，因此暂存变更不会改变该列表。请在外部 Git 客户端中解决合并冲突。

![已更改的文件](images/workflow/changed_files.png)

只选择一个已修改或已重命名的文件，然后单击 <kbd>Diff</kbd> 查看其文本差异。单击 <kbd>Revert</kbd> 会丢弃所选文件在工作树和索引中的变更。已跟踪文件会恢复到 `HEAD`；不存在于 `HEAD` 中的文件会被删除，无论它们是否作为添加项暂存；重命名操作会删除新路径并恢复旧路径。无法在编辑器中撤销此操作，因此请提交或备份以后可能需要的工作。

## Git

Git 可高效存储 Defold 基于文本的项目文件。频繁变化的大型二进制资源，例如 PSD 或音频制作文件，仍可能使仓库历史迅速增长。可考虑对大型工作文件使用 Git LFS，或使用单独的存储和备份解决方案。

*Changed Files* 面板只提供本地状态、差异和恢复操作。它不知道提交是否已推送到远程仓库，也不会获取、拉取、提交或推送变更。请在外部 Git 客户端或命令行中执行这些操作。默认情况下，Defold 会在重新获得焦点时加载外部变更并刷新该面板。如果已禁用 *Load External Changes on App Focus*，请选择 *File ▸ Load External Changes*。
