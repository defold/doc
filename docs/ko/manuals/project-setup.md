# 프로젝트 준비

Defold 에디터 안에서 새 프로젝트를 쉽게 생성할 수 있습니다. 컴퓨터에 이미 있는 기존 프로젝트를 여는 옵션도 있습니다.

## 새 로컬 프로젝트 생성하기

<kbd>New Project</kbd> 옵션을 클릭하고 생성하려는 프로젝트 종류를 선택합니다. 프로젝트 파일을 저장할 하드 드라이브 위치를 지정합니다. 선택한 위치에 프로젝트를 생성하려면 <kbd>Create New Project</kbd>를 클릭합니다. Template을 사용해 새 프로젝트를 생성할 수 있습니다:

![프로젝트 열기](images/workflow/open_project.png)

또는 단계별 지침이 있는 Tutorial에서 생성할 수 있습니다:

![튜토리얼에서 프로젝트 생성하기](images/workflow/create_from_tutorial.png)

또는 완성된 Sample 게임에서 생성할 수 있습니다:

![샘플에서 프로젝트 생성하기](images/workflow/create_from_sample.png)

### 프로젝트를 GitHub에 추가하기

로컬 프로젝트는 어떤 버전 관리 시스템과도 연동되어 있지 않으므로, 파일은 하드 드라이브에만 존재합니다. 파일에 적용한 변경 사항은 영구적이며 변경 사항을 되돌리거나 삭제된 파일을 복구할 방법이 없습니다. Git 같은 버전 관리 시스템을 사용하여 파일 변경 사항을 추적하는 것이 좋습니다. 이렇게 하면 다른 사람들과 프로젝트에서 공동작업하기도 매우 쉬워집니다. 로컬 프로젝트를 GitHub에 업로드하는 과정은 몇 단계만으로 완료할 수 있습니다:

1. [GitHub](https://github.com/)에서 계정을 생성하거나 로그인합니다
2. [New Repository](https://help.github.com/en/articles/creating-a-new-repository) 옵션을 사용해 저장소(repository)를 생성합니다
3. [Upload Files](https://help.github.com/en/articles/adding-a-file-to-a-repository) 옵션으로 모든 프로젝트 파일을 업로드합니다

이제 프로젝트는 버전 관리 아래에 있으므로, [프로젝트를 클론](https://help.github.com/en/articles/cloning-a-repository)하여 로컬 하드 드라이브로 가져오고 앞으로는 이 새 위치에서 작업해야 합니다.

## 기존 프로젝트 열기

컴퓨터에 이미 있는 프로젝트를 열려면 <kbd>Open From Disk</kbd> 옵션을 클릭합니다.

![프로젝트 임포트](images/workflow/open_from_disk.png)

## 최근 프로젝트 열기

프로젝트를 한 번 열면 최근 프로젝트 목록에 표시됩니다. 이 목록에는 가장 최근에 작업한 프로젝트가 표시되며, 목록에서 프로젝트를 더블 클릭하면 원하는 프로젝트를 빠르게 열 수 있습니다.
