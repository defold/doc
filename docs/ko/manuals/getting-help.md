# 도움 받기

Defold를 사용하는 동안 문제가 발생하면 알려 주세요. 문제를 수정하거나 우회할 수 있도록 도와드릴 수 있습니다! 문제를 논의하고 보고할 수 있는 방법은 여러 가지가 있습니다. 가장 적합한 옵션을 선택하세요:

## 포럼에서 문제 보고하기

문제를 논의하고 도움을 받는 좋은 방법은 [포럼](https://forum.defold.com)에 질문을 게시하는 것입니다. 발생한 문제 유형에 따라 [Questions](https://forum.defold.com/c/questions) 또는 [Bugs](https://forum.defold.com/c/bugs) 카테고리에 게시하세요. 질문이나 이슈를 올리기 전에 이미 해결 방법이 있을 수 있으므로 [검색](https://forum.defold.com/search)해 보세요.

질문이 여러 개라면 게시물을 여러 개 만드세요. 관련 없는 질문을 같은 게시물에서 묻지 마세요.

### 필요한 정보
필요한 정보를 제공하지 않으면 지원을 제공할 수 없습니다:

**제목**
짧고 설명적인 제목을 사용하세요. 좋은 제목의 예는 "회전한 방향으로 게임 오브젝트를 이동하려면 어떻게 해야 하나요?" 또는 "스프라이트를 페이드 아웃하려면 어떻게 해야 하나요?"입니다. 나쁜 제목의 예는 "Defold를 사용하는 데 도움이 필요합니다!" 또는 "게임이 작동하지 않습니다!"입니다.

**버그 설명(필수)**
버그가 무엇인지 명확하고 간결하게 설명합니다.

**재현 방법(필수)**
동작을 재현하는 단계:
1. '...'로 이동합니다
2. '....'를 클릭합니다
3. '....'까지 아래로 스크롤합니다
4. 오류를 확인합니다

**예상 동작(필수)**
발생할 것으로 예상한 내용을 명확하고 간결하게 설명합니다.

**Defold 버전(필수):**
  - 버전 [예: 1.2.155]

**플랫폼(필수):**
 - 플랫폼: [예: iOS, Android, Windows, macOS, Linux, HTML5]
 - OS: [예: iOS8.1, Windows 10, High Sierra]
 - 기기: [예: iPhone6]

**최소 재현 케이스 프로젝트(선택 사항):**
버그가 재현되는 최소 프로젝트를 첨부하세요. 버그를 조사하고 수정하려는 사람에게 큰 도움이 됩니다.

**로그(선택 사항):**
엔진, 에디터 또는 빌드 서버의 관련 로그를 제공하세요. 로그 저장 위치는 [여기](#log-files)에서 확인할 수 있습니다.

**우회 방법(선택 사항):**
우회 방법이 있다면 여기에 설명하세요.

**스크린샷(선택 사항):**
해당하는 경우 문제를 설명하는 데 도움이 되는 스크린샷을 추가하세요.

**추가 컨텍스트(선택 사항):**
문제에 관한 다른 컨텍스트가 있으면 여기에 추가하세요.


### 코드 공유하기
코드를 공유할 때는 스크린샷이 아니라 텍스트로 공유하는 것이 좋습니다. 텍스트로 공유하면 검색하고, 오류를 강조 표시하고, 수정 사항을 제안하고 적용하기 쉽습니다. 코드는 세 개의 \`\`\`로 감싸거나 4칸 들여쓰기하여 공유하세요.

예:

\`\`\`
print("Hello code!")
\`\`\`

결과:

```
print("Hello code!")
```


## 에디터에서 문제 보고하기

에디터는 이슈를 편리하게 보고할 수 있는 방법을 제공합니다. 이슈를 보고하려면 에디터 안에서 <kbd>Help->Report Issue</kbd> 메뉴 옵션을 선택합니다.

![](images/getting_help/report_issue.png)

이 메뉴 옵션을 선택하면 GitHub의 이슈 트래커로 이동합니다. [로그 파일](#log-files), 운영체제 정보, 문제를 재현하는 단계, 가능한 우회 방법 등을 제공하세요.

::: sidenote
이 방법으로 버그 리포트를 제출하려면 GitHub 계정이 필요합니다.
:::


## Discord에서 문제 논의하기

Defold를 사용하는 동안 문제가 생기면 [Discord](https://www.defold.com/discord/)에서 질문해 볼 수 있습니다. 다만 복잡한 질문과 심층 논의는 포럼에 게시하는 것을 권장합니다. 또한 Discord를 통해 제출된 버그 리포트는 접수하지 않는다는 점에 유의하세요.
