# 물리

Defold에는 2D 물리 시뮬레이션용 [Box2D](https://box2d.org/)와 3D 물리용 Bullet이 포함되어 있습니다. [App Manifest의 Physics 2D 설정](/manuals/app-manifest/#physics-2d)에서 **Box2D Version 3**, **Box2D (Legacy Defold version)** 또는 **None**을 선택합니다. 레거시 구현이 기본값이고 Box2D 3은 선택적으로 활성화합니다. 구현을 바꾸면 시뮬레이션 결과가 달라질 수 있으며 버전별 [Box2D 프로젝트 설정](/manuals/project-settings/#box2d)을 다시 조정해야 할 수 있습니다.

이 매뉴얼에서 설명하는 컴포넌트 중심의 충돌 오브젝트 워크플로와 `physics` 모듈은 두 Box2D 구현 모두에서 동작합니다. **None**을 선택하면 2D 물리가 제거됩니다. 또한 Defold는 2D body, shape, joint, chain, world에 직접 액세스할 수 있는 저수준 [`b2d`](/ref/stable/b2d/), `b2d.body`, `b2d.fixture`, `b2d.shape`, `b2d.joint`, `b2d.chain`, `b2d.world` API를 제공합니다. 모든 저수준 함수를 두 Box2D 구현에서 사용할 수 있는 것은 아니므로 각 함수의 생성된 API 문서를 App Manifest에서 선택한 구현과 대조해 확인하세요.

Defold에서 사용하는 물리 엔진의 주요 개념은 다음과 같습니다:

* **충돌 오브젝트** - 충돌 오브젝트는 게임 오브젝트에 물리 동작을 부여하는 데 사용하는 컴포넌트입니다. 충돌 오브젝트에는 무게, 마찰, 형태 같은 물리 속성이 있습니다. [충돌 오브젝트를 만드는 방법을 알아보세요](/manuals/physics-objects).
* **충돌 모양** - 충돌 오브젝트는 공간상의 범위를 정의하기 위해 여러 기본 모양 또는 하나의 복잡한 모양을 사용할 수 있습니다. [충돌 오브젝트에 모양을 추가하는 방법을 알아보세요](/manuals/physics-shapes).
* **충돌 그룹** - 모든 충돌 오브젝트는 미리 정의된 그룹에 속해야 하며, 각 충돌 오브젝트는 충돌할 수 있는 다른 그룹 목록을 지정할 수 있습니다. [충돌 그룹을 사용하는 방법을 알아보세요](/manuals/physics-groups).
* **충돌 메세지** - 두 충돌 오브젝트가 충돌하면 물리 엔진은 해당 컴포넌트가 속한 게임 오브젝트에 메세지를 보냅니다. [충돌 메세지에 대해 더 알아보세요](/manuals/physics-messages)

충돌 오브젝트 자체 외에도 두 충돌 오브젝트를 연결하고, 움직임을 제한하거나 힘을 적용하는 방식으로 물리 시뮬레이션에서의 동작에 영향을 주는 충돌 오브젝트 **제약 조건**을 정의할 수 있습니다. 이는 보통 **조인트**라고 부릅니다. [조인트에 대해 더 알아보세요](/manuals/physics-joints).

또한 **레이 캐스트**라고 하는 선형 광선을 따라 물리 월드를 검사하고 읽을 수 있습니다. [레이 캐스트에 대해 더 알아보세요](/manuals/physics-ray-casts).


## 물리 엔진 시뮬레이션에서 사용하는 단위

물리 엔진은 뉴턴 물리를 시뮬레이션하며 미터, 킬로그램, 초(MKS) 단위에서 잘 동작하도록 설계되어 있습니다. 또한 물리 엔진은 움직이는 오브젝트가 0.1미터에서 10미터 사이의 크기일 때 잘 동작하도록 조정되어 있으며(정적 오브젝트는 더 클 수 있습니다), 기본적으로 엔진은 1단위(픽셀)를 1미터로 취급합니다. 픽셀과 미터 사이의 이러한 변환은 시뮬레이션 수준에서는 편리하지만, 게임 제작 관점에서는 그다지 유용하지 않습니다. 기본 설정에서는 크기가 200픽셀인 충돌 모양이 200미터 크기로 취급되며, 이는 적어도 움직이는 오브젝트에는 권장 범위를 크게 벗어납니다.

일반적으로 게임에서 흔히 사용하는 오브젝트 크기와 잘 맞도록 물리 시뮬레이션을 스케일링해야 합니다. 물리 시뮬레이션의 스케일은 *game.project*의 [물리 스케일 설정](/manuals/project-settings/#physics)에서 변경할 수 있습니다. 예를 들어 이 값을 0.02로 설정하면 200픽셀이 4미터로 취급됩니다. 스케일 변경에 맞추려면 중력(*game.project*에서도 변경 가능)도 함께 증가시켜야 한다는 점에 유의하세요.


## 물리 업데이트 {#physics-updates}

안정적인 시뮬레이션을 위해 물리 엔진은 일정한 간격으로 업데이트하는 것이 좋습니다(프레임 속도에 따라 불규칙해질 수 있는 간격으로 업데이트하는 것과 반대입니다). *game.project* 파일의 Physics 섹션에서 [Use Fixed Timestep 설정](/manuals/project-settings/#physics)을 체크하면 물리에 고정 업데이트를 사용할 수 있습니다. 업데이트 빈도는 *game.project* 파일의 Engine 섹션에 있는 [Fixed Update Frequency 설정](/manuals/project-settings/#engine)으로 제어합니다. 물리에 고정 타임스텝을 사용할 때는 게임의 충돌 오브젝트와 상호작용하기 위해, 예를 들어 힘을 적용할 때, `fixed_update(self, dt)` 라이프사이클 함수를 사용하는 것도 권장됩니다.


## 주의 사항 및 일반적인 문제

컬렉션 프록시
: 컬렉션 프록시를 사용하면 둘 이상의 최상위 컬렉션, 즉 *게임 월드*를 엔진에 로드할 수 있습니다. 이 경우 각 최상위 컬렉션은 별도의 물리 월드라는 점을 알아야 합니다. 물리 상호작용([충돌, 트리거](/manuals/physics-messages) 및 [레이 캐스트](/manuals/physics-ray-casts))은 같은 월드에 속한 오브젝트 사이에서만 발생합니다. 따라서 두 월드의 충돌 오브젝트가 시각적으로 서로 정확히 겹쳐 있더라도 물리 상호작용은 발생하지 않습니다.

충돌이 감지되지 않음
: 충돌이 제대로 처리되거나 감지되지 않는 문제가 있다면 [디버깅 매뉴얼의 물리 디버깅](/manuals/debugging-game-logic/#debugging-problems-with-physics)을 읽어보세요.
